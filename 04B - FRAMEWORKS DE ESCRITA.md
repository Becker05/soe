# 04B — FRAMEWORKS DE ESCRITA (como executar)

Toda estrutura termina reforçando um ativo da marca e responde "como isso melhora uma decisão do gestor?".

## Proprietários (canônico = skill `conteudo-low-ticket`)

PRSA · PRS · CRIEI — definição oficial na skill. Aqui apenas referenciados, para não duplicar. Se editar, edite na skill.

## Diagnóstico

Situação → Pergunta → Explicação → Impacto → Decisão → CTA.
Ex.: "Seu ROAS caiu." → "Você aumentaria o orçamento?" → "Depende." → "Porque..." → "A decisão correta é..." → CTA.

## Quebra de Paradigma

Todo mundo acredita em X → na prática acontece Y → o problema é... → a decisão correta é... → CTA.
Ex.: "Todo mundo compara Meta com GA4." → "Mas não respondem à mesma pergunta." → "Por isso..." → CTA.

## Auditoria

Print → problema encontrado → por que aconteceu → impacto → como foi corrigido → resultado. Sem mostrar configuração; mostrando decisão.

## Pergunta

Pergunta → hipótese → resposta → explicação → decisão → CTA.
Ex.: "Essa campanha realmente vendeu?" → "Talvez." → "Veja..."

## PAS

Problema → Agitação (consequência de decidir sem dado confiável) → Solução (a estrutura de mensuração). Fecha na Cartilha.

## Storytelling

Caso real (anonimizado) → o que os números diziam antes → a virada na leitura → a decisão que mudou → resultado. Herói é a mensuração, nunca a ferramenta.

## Mito

"Todo mundo acha que X." → por que leva a decisão errada → o que os dados confiáveis mostram → nova decisão. Corrigir a leitura, nunca ridicularizar a plataforma.

## Comparação

Duas leituras do mesmo dado → por que divergem → qual decisão cada uma leva → como a estrutura correta reconcilia.

## Lista

"N sinais de que sua mensuração está te fazendo decidir no escuro." Cada item = sintoma + decisão que ele compromete.

## Implementação (antes "Tutorial")

Passo a passo só quando serve a uma decisão. Enquadrar como "organizar sua mensuração para X decisão", nunca como "curso de GA4". (Nome "Implementação" lembra consultoria; "Tutorial" lembra YouTube.)

---

## Vídeo curto

Não substitui os frameworks acima. É a estrutura de tempo que os carrega quando a peça é vídeo de até 30s. O framework narrativo continua sendo escolhido pelo 04A; este define o ritmo da entrega.

0–2s · gancho falado no primeiro frame, sem saudação e sem logo
2–4s · mudança visual
5–15s · prova na tela
15–23s · conclusão que muda uma decisão
23–28s · CTA

Regras: cortes a cada 1,5–2,5s · legenda queimada · o número aparece na tela, não só na fala · a prova precisa estar marcada como exibível no verbete (campo "Prova exibível", doc 06).

Base: padrões de pesquisa de 07/2026 e D-05 (doc 09).

## Peça citável

Para conteúdo desenhado para ser repostado ou citado por gestor e agência, não para ser consumido só por quem já segue.

Requisitos: se sustenta fora do contexto do perfil · carrega um número que sobrevive ao recorte · não depende de peça anterior · o CTA não pressupõe que o leitor já segue. Fecha na tese, não no pedido.

Base: D-03 (doc 09).
