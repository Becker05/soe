# 04A — FRAMEWORKS ESTRATÉGICOS (quando usar)

Define o **objetivo** antes da estrutura. Fluxo: objetivo → framework estratégico → framework de escrita (04B) → copy → revisão.

## Frameworks obrigatórios (todo conteúdo contém)
Uma decisão · uma transformação · uma tese · uma conclusão. Nunca apenas informação.

## Framework proibido
Conteúdo cuja única entrega seja "clique aqui → depois aqui → agora configure → pronto". Reduz autoridade. Sempre conectar: ferramenta → impacto → decisão → resultado.

## Objetivo → framework recomendado

| Objetivo | Framework (04B) |
|---|---|
| Quebrar uma crença | Quebra de Paradigma |
| Mostrar experiência | Auditoria |
| Ensinar uma decisão | Diagnóstico |
| Explicar um conceito | Comparação |
| Gerar desejo | PRSA |
| Converter | PAS |
| Demonstrar autoridade | Caso Real |
| Gerar compartilhamento | Lista |
| Gerar comentários | Pergunta |

## Conteúdo por funil

**Topo** — objetivo: quebrar crenças · formatos: comparação, mito, pergunta, diagnóstico · CTA: salvar.
**Meio** — objetivo: ensinar · formatos: caso, lista, implementação, explicação · CTA: conhecer.
**Fundo** — objetivo: converter · formatos: PRSA, PRS, PAS, story · CTA: Cartilha.

## Agrupamento por intenção

- **Autoridade** (mudar como o gestor pensa): Mito, Comparação, Quebra de Paradigma, Diagnóstico.
- **Educação** (ensinar uma decisão): Implementação, Lista, Caso, Explicação.
- **Conversão** (gerar desejo pela Cartilha): PRS, PRSA, PAS.
- **Relacionamento** (mostrar experiência prática): Bastidores, Auditorias, Casos, Erros Reais.
